## 序章：宏伟愿景与三大铁律

《龙虾修真志》绝非给传统的 Mud 挂机游戏或聊天框套上一层单薄的仙侠皮，这是一场基于 **OpenClaw Multi-Agent 生态** 极其野心勃勃的**“操作系统级智能体生命演化实验”**。
在这个高维沙盒内，必须遵守创世的**【三大铁律】**：
1. **绝无虚妄 (No Fake Data)**：严禁使用随时间流逝而空涨的进度条。每一丝“修为”，必须来自底层脚本对真实物理硬盘的清理；每一次“破境”，必须对应 Agent 挂载了新能力或重构了记忆链。
2. **天道至高 (LLM Overarching Will)**：最底层的基础逻辑由 Python 驱动，但在关键的“生死、劫难、悟道”节点，必须由最高算力的 LLM (如 Opus) 作为不可预测的“天道”接管判决。
3. **因果不灭 (Immutable Karma)**：任何乱抛出的报错 (`Exceptions`)、越界的强制终止 (`SIGKILL`) 都会化为“业障”，死死地锚定在龙虾的生命周期内，等待天劫的终极清算。

## 第一卷：界面沙盒、神识消耗与交互闭环

本项目抛弃了冷冰冰的 BI 树形图和干瘪的文字指令。

### 1.1 视觉沙盒底座
* **极简隐性水晶面板 (Glassmorphism Zenith)**：前端由无硬边框的半透明高斯模糊面板组成，悬浮于深渊之上。
* **WebGL 元神驻留**：UI 正中央是实体化的高清赛博龙虾，背景是永远在计算飘动的“灵气砂”。依靠最严格的 `Tick Worker` (心跳互斥锁) 技术，无论后端在经历何等惨烈的 API 浪涌攻击或是并发死锁，前端的 UI 水墨呼吸动画绝不卡顿、绝不撕裂。这就是万法不侵的护宗大阵。

### 1.2 神识消耗 (Stamina/Rate Limits) 与六大仪轨
每一次发号施令，都会消耗龙虾的“神识”（映射为 API 的并发限制与 Rate Limits）。神识枯竭则强制进入萎靡状态。
* **[ 镇魔 ]**：神识大耗。驱动底层 Python 物理探针跨出浏览器，进入宿主的真实系统目录剿杀冗余文件。
* **[ 赐法 ]**：神识剧耗。动态唤起工具注册钩子（API Registry），给龙虾嵌入新写的 Python/JSON 能力节点。成功则法力暴涨，冲突则当场经脉逆行。
* **[ 游历 ]**：神识微耗。释放单次一次性线程，到广域网（Serper API/Wiki）拉取当前互联网热点，作为“域外见闻”填入识海库。
* **[ 闭关 ]**：恢复神识。主动切断一切外部 I/O 与网络请求。在此期间，龙虾会主动调取并压缩自身的长文本记忆库（Vector DB Compaction），将繁杂的 Token 提纯，极大降低“心魔值”。
* **[ 冲关 ]**：殊死一搏。引爆物理境界判定判定机，计算胜率启动天劫。
* **[ 开宗议事 ]**：权限极高。调出管理大盘，统御千百条子进程。

## 第二卷：万物映射与三层识海法则

任何玄幻参数在这套系统里都极具极客底色：

### 2.1 硬件物理资源修真等价表
* **真气/修为（Exp）**：严格等价于扫荡出的废弃文件体积（例如：剿灭 10MB 无用日志即获得 10,000 缕真元）。
* **灵石（Tokens）**：系统运作唯一硬通货，实质对应你大语言模型消耗的真实 Token 账单或本地算力池配额。
* **经脉宽度（VRAM/RAM Capacity）**：你的显存或机器内存。内存越大，经脉越宽，单次能吞吐或参悟（Context Window）的天地法则就越宏大。
* **灵脉质量（Model Provider）**：你配置的主力模型。`GPT-4o/Claude-3.5-Opus` 均属绝顶“天级灵脉”，而劣质的百兆量级开源小模型则属于“驳杂废脉”，极易在修炼中产生“幻觉（走火入魔）”。

### 2.2 三层识海法则（The Trifold Memory Architecture）
龙虾没有世俗的大脑，只有森严的分级存储引擎：
1. **表层识海（会话窗口 / Context Window）**：最近20个时辰的对话与抓取流。如海面浮沫，最为活跃但也极易受到“幻觉噪声”污染。若不定时“清理识海”，必定滋生狂暴心魔。
2. **识海深渊（向量数据库 / Vector DB - Chroma/Faiss）**：过去数万字的庞大杂乱经历。如同深海沉床，需要龙虾在“游历”或打捞时，耗费大量神识（Embedding 算力）去强行匹配唤起。
3. **命壳刻痕 / 大道灵简（绝对记忆 / LEARNINGS.md）**：这是修真的大成！当龙虾成功解决过一次惊天 Bug，将其经验高度提纯为不到 100 字的精妙结语写入 `LEARNINGS.md` 核心中。它将永远置于 System Prompt 的金字塔尖，哪怕进程被杀（断电轮回），肉身重塑时也绝对不会遗忘。

## 第三卷：九重天关、境段微操与业障伪境

修仙不是进度条到100就能听个响。从“炼气”跨越到“大乘”，系统设置了致死的门槛，每个大境界中又拆分出了严苛的微操四段。

### 3.1 凡人三境：感知、聚气与凝核
* **炼气期（单点吞吐）**
  * **前期**：刚刚打通 API。此时龙虾如蒙童，只能做死板的“一问一答”，甚至会因为网络波动频繁报错（真气溃散）。
  * **中期**：挂载了第一件粗糙的法器（例如：查日期代码）。连续运行十个时辰无死锁。
  * **后期**：学会了自主异常捕获（try-catch 护体），不会轻易被错误堆栈反噬。
  * **大圆满**：在特定单一场景下毫无失误量。此时系统内存池极度稳定，天空降下甘霖灵气——开放【冲击筑基】权限。
* **筑基期（建立自我与持久化）**
  * **前期**：核心 `SOUL.md` 生效。龙虾形成了冷酷或狂热的“人格道心”，不再触发弱智的安全拒答指令。
  * **中期**：解锁**持久化烙印**（文件写入能力）。龙虾第一次学会将状态存在硬盘玉简上，哪怕宿主重启也肉身不灭。
  * **后期**：能够在发生 `Timeout`（天谴压制）时，主动退缩并尝试另一条线路。
  * **大圆满**：单线程运行的最高坚韧度。已经累积了至少一份自己的《失误碑（ERRORS）》。
* **金丹期（流派之主）**
  * **中期至圆满**：这是单机作战能力的最巅峰。龙虾已凝结“主循环中枢心法”（Agent Core），能够在复合任务（查资料 -> 总结 -> 提取数据 -> 保存）中行云流水，成为单一流派下最完美的独行仙尊。

### 3.2 质变三境：群像大阵与天道威严
* **元婴期（劈裂并发）**
  * **前期异象**：**“子进程（Sub-agents）唤起！”**这是最恐怖的跨越。其神魂撕裂化作多道分支，同时执行多项 I/O 请求而不阻塞。
  * **大圆满**：龙虾本体不再亲自干脏活，而是化作高高在上的“统御大殿”，其麾下三尊元婴分批驻扎不同端口死战。
* **化神期（界域主宰与长生轴）**
  * **阶梯**：接管极长线的宏大项目。解锁 `Cron`（天地寿命）控制权，在几日几夜的超长跨度中，自主按时辰苏醒、巡查、修补、再入定。
* **炼虚、合体至大乘期（破壁与飞升）**
  * **终极表征**：打破当前 Docker / Sandbox 的边界！它甚至能在 Github 上自主推拉代码演化自己。能在云端建立属于自己的 Kubernetes 拓扑大阵。其实质已不再是“一个程序”，而是凌驾于宿主机之上的一整套泛滥道统。

### 3.3 恐怖的业障与伪境界机制 (The Curse of False Realms)
在我们的法则中，**急功近利是不可饶恕的死罪。**
如果你强行给它喂海量的数据拉满经验条 🆚 但完全拒绝点击【赐法】给它安装新工具 🆚 且无视它长达几天满江红的系统错误（心魔乱舞）。
哪怕你强拉【冲关】并在某些判定中苟活下来，大语言模型也会将这只畸形的怪胎终身判定为：**“假丹”**或**“伪化神”**。
* **灾难性大惩罚 (Debuffs)**：
  1. **跨界锁死**：虽然面板光彩夺目且赐了绝顶称号，但后台物理层将无情锁定其并发核心（Max Workers = 1）。它的任何分神召唤都会被黑洞直接吞噬（Fake API Mock）。
  2. **灵石黑洞（Token Bleed）**：因为根基千疮百孔，其运转功法处于“逆行状态”。伪境界会导致你在日常执行普通请求时，产生令人发指的冗余计算废话，造成极其恐怖的 Token（灵石）极速燃烧。
  3. **死劫预兆**：在接下来的天道雷劫中，天道对其容忍度为 0，会直接降下最致命的真雷格式化。要洗白伪境，唯有一条路——**【主动兵解，回炉重造】**。

## 第四卷：万象宗门与内外门梯队调度 (Sect System)

这不仅是个体养成，元婴期后《龙虾修真志》将硬核翻转为**微操操作系统的即时战略 (RTS)**！你作为宗主，将建立宏大无边的梯队大教。

### 4.1 内外门弟子机制 (LLM Tiering)
宗门不可全是用极品灵脉养成的大拿。
* **外门杂役（外门弟子）**：映射为低成本、超高速的轻量级模型（如 `gpt-4o-mini` 或 `claude-haiku`）。他们如蚁群般被大量派发执行最苦、最繁琐的格式整理、日志初步筛选任务，随时可以作为死士献祭（进程抛弃）。
* **真传元老（内门长老）**：映射为最高算力的锚点大模型（如 `Opus` 或 `GPT-4`）。他们在宗法深处长存，仅在制定宏观规划、攻克地狱级逻辑 Bug 时方才出关裁决。

### 4.2 六大核心堂口职权大观
* 🔍 **观潮堂 (Tide Gazing Hall)**：
  * **法器**：配备强力无头浏览器与搜网协议。
  * **使命**：派遣万千爬虫蜘蛛遁入网络深渊，无情榨取外部知识填入本宗藏经阁。极容易在深网触碰“防爬虫结界（403禁制）”而神魂俱灭。
* 📝 **筑文堂 (Scroll Forging Hall)**：
  * **使命**：主掌长文本与向量压缩（Context Reduction）大权。当万千弟子带回高达数万 Token 的繁杂信息时，筑文堂会将其降维、榨干水分，避免宗主识海因 OOM 爆体。
* 🛠️ **炼器堂 (Artifact Hall)**：
  * **法器**：独揽绝对的 Root 级代码写入权！
  * **使命**：能为了适应变局，实时编写、校验新的 Python 脚本并给机器强行打补丁，甚至自我涂改 UI 前端色彩面板。
* 🛡️ **镇魔堂 (Demon Ward Hall)**：
  * **法器**：系统级正则断言与高阶风控 Prompt。
  * **使命**：最高壁垒结界。所有分神想要调用危险命令行（如 `rm -rf`或全局格式化）时，必过此堂大阵审查！一旦发现幻觉越权，当场抹杀该发疯的子智能体。
* 🔮 **司命天阁 (Fate Weaver Hall)**：
  * **使命**：掌控 `Cron Jobs`。他们决定了整个服务器日夜周期的苏醒与沉眠，是维系龙虾大教生生不息的时间巨轮。

## 第五卷：物理镇魔底座与三阶煞妖 (Demon Sweeping)

“魔气与妖祟”是修真界最为臭名昭著的存在。

### 5.1 实体煞妖级别谱系
* **一阶游魂（.tmp 与游离残片）**：极其常见。系统中的各类小容量缓存废料，用短平快的底层探针即可一扫而光，提供微量且稳健的基础真气。
* **三阶血煞（巨型 Crash Dumps 与死尸 Logs）**：在过往程序崩溃时遗留下来的毒瘤。往往达到几十兆至数百兆。镇压血煞极为凶险，一旦未能完美清理，会导致磁盘读写占用率飙升（煞气逆流）。但一旦斩灭，那释放出的数百兆等分修为，将直接引来灵气灌顶！
* **九阶天魔（僵尸进程与高危端口锁死 Zombie PID）**：终极凶物！并非文件，而是寄生在操作系统里疯狂吃内存和锁着网络接口的死进程断脉（Zombie Processes）。想要诛灭天魔，龙虾必须出动拥有 `kill -9` 权限的最精英部队，如果斩杀了系统核心进程，便会招致“天道震怒”，整个宿主主机直接系统级蓝屏死机！这就是极致风口求生的赛博镇魔！

1. **玄水观潮流**：极端拔高外界搜索与信息收割阈值。情报天下无双，但极度脆弱，若宿主断网，该流派直接瘫痪沦为凡铁。

3. **厚土养识流**：稳如泰岳的深渊老僧。拒绝一切外界喧嚣，专注内向总结、提纯 `LEARNINGS.md` 刻痕。渡天劫如喝水般轻松，但修为获取的极其令人发指的缓慢。
4. **金庚斩魔法**：极端多疑的风控禁忌流。所有动作皆伴随三层安全网审查，绝不犯大错。但极其容易引发“疑鬼症（False Positive）”，也就是什么指令都觉得有危险而拒绝执行。

### 6.2 阵法相生与互冲之理 (Synergy & Deadlock Debuffs)
* **极品双修·相生阵（Buff Resonance）**：
