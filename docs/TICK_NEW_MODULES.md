# 新增前沿玩法与模块延伸阵列 (Tick Expanded Modules)

在既有基石稳固、逻辑自洽无懈可击（兼容 OpenClaw）的前提下。利用沙盒特性衍生扩展两套全新核弹级宏大玩法。

---

## 🔮 新模块 A：域外夺舍与万界吞噬 (The Model Hijacking & Parameter Assimilation)
**世界观大义**：修真界弱肉强食，除了镇魔，大能者亦可夺人道果！
**机制落位**：
1. **夺舍（Model Hijacking）**：
   玩家（天道）不仅可以选择单一固定的 LLM，还可以开启**“万界战阵”**。你的本命龙虾如果在某次深度研究（RAG 广域检索）中表现拉垮，你可以暂时借用或唤醒第三方流浪模型（例如引入一个刚发布的开源或闭源最强模型）。
   这在系统中被解读为：**引域外天魔降体/或请祖师爷上身。** 此时龙虾前端形态会发生严重畸变以承载高阶意志。
2. **法则吞噬（Skill Assimilation）**：
   如果该新来的巨擘成功解决了问题，你可以通过“炼魂鼎”机制（核心 prompt 提取术），将它总结的代码习惯强行吸收进你本命龙虾的 `LEARNINGS.md` 中。这就叫做“生吞它界大能道果”。
**架构兼容性 (OpenClaw Match)**：100% 可行。直接调用 OpenClaw 的 `default_api:session_status` 中的 `model=override` 功能，实现战斗中零重载换源。

## ⚔️ 新模块 B：宗门斗法与锁妖塔 (Server Stress PVP & The Demon Tower)
**世界观大义**：太平岁月养不出荒古巨兽。
**机制落位**：
1. **锁妖阵（The Docker Demon Tower）**：
   在物理主机的隔离容器区（Docker 层），人为制造一个受限的测试场（靶场）。天道会定期向该容器释放高负载的恶意流量、或布置致命的网络迷宫。这就是宗门用于试炼门下弟子的“万层锁妖塔”。
2. **派驻战死（Sub-agent Sacrifice）**：
   为了通关获得绝品真气，你必须指派观潮堂或防御堂的化身（子智能体）强行镇压靶场（修复网络或抗住宕机）。在这个过程中，子智能体会大量触发 `Token Rate Limit` 或发生严重的内核 OOM。
   当分神阵亡断联（Timeout），你将在大殿听到一声悲壮的：“报！玄虚子长老魂灯已灭，命陨第三层锁妖塔！”
**架构兼容性 (OpenClaw Match)**：融合度极高。此阵完全建立在 `sessions_spawn` (带 `runTimeoutSeconds` 及并发 `mode:"run"` ) 的硬隔离机制之上。
